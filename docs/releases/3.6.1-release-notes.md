# Forgeloop 3.6.1

Forgeloop 3.6.1 修正 Codex 中 Skill 名称与显示名称不一致的问题，并把插件运行包与仓库开发工具彻底分离。安装后的插件现在只包含清单和 Skills，不再携带生成配置、Fixture、维护脚本、测试或系统杂项文件。

## Skill 名称一致

- 20 个 Skill 的 `display_name` 现在与正式 Skill 名称完全一致；
- Codex 界面中的 `to-spec`、`to-tickets` 等入口不再显示为容易误解的动作别名；
- 发布校验会拒绝显示名称与 Skill 标识漂移，避免同类问题再次进入正式版本。

## 运行包边界收敛

- `plugins/forgeloop/` 只保留 `.codex-plugin/` 与 `skills/`，对应 Codex 真正需要安装的内容；
- 生成配置、Fixture、维护脚本和测试统一迁移到 `tooling/forgeloop/`；
- Release Validator 对插件顶层执行严格白名单检查，拒绝开发目录、`.DS_Store` 或其他杂项文件进入安装缓存；
- README 中的维护命令已切换到新的仓库根目录入口。

## 评估契约修正

- Agent evaluation 明确区分旧 Run 的即时动作与未来新 Spec/Run 的生命周期；
- Acceptance Repair 的输出只描述修复前的当前路由，不把修复完成后的最终重新验收误记为立即重跑；
- 期望动作与产品协议保持不变，只消除评估字段在作用对象和时间点上的歧义。

## 兼容性

- 保持 20 个正式 Skill、11 个用户入口和 9 个模型可调用能力不变；
- 不新增运行时状态、结果枚举、Tracker 状态或配置迁移；
- 现有 Spec、Ticket、Run 与历史证据无需迁移。

## 验证

- Release Validator、Runtime Contract、Skill UI 元数据、上游同步和 Plugin Validator；
- Python 全量单元测试与 authenticated Codex Agent evaluation；
- 本地 Marketplace 候选重装、Installed Validator 与安装目录边界回读；
- Release Tag、GitHub Release 与最终 `main` Commit 精确绑定。
