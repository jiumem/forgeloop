# 自动修复周期以进展约束而非固定次数终止

修复周期不设置任意的总数上限：只要工作仍在原 Ticket Scope 内、上一机制已有证伪证据、存在实质不同且可证伪的新机制，并且 Candidate 或可观察证据产生真实进展，Scheduler 就可以自动续配而不请求用户批准。需要改变契约时转 `CONTRACT_BLOCKER`，没有新机制或没有进展时转 `IMPLEMENTATION_BLOCKED`，环境与权限故障、取消及恢复冲突继续使用各自已有出口；宿主资源限制与实现正确性的修复周期保持分离。新机制与真实进展由 Agent 基于完整契约、历史 Findings、Candidate 和证据进行语义判断；可以使用稳定、清晰的语义字段组织理由、证据和下一步，供 Agent 阅读、比较、恢复与审查，但这些字段不构成需要 parser、正则、刚性枚举或穷举转换表才能执行的第二套程序化状态机，也不得替代 Agent 对整体语义的一致性判断。
