# Forgeloop 3.1.0

Forgeloop 3.1.0 在保持 20 个 Skill 与 Tracker 驱动架构不变的前提下，收紧交付验收、风险评审、修复诊断、Checkpoint 传输与累计审计契约。

## 主要改进

- 建立唯一的 `Delivery Acceptance` 事实源，并将它与发布边界明确分离；
- 以既有 Validation Entry 承载跨 seam 不变量证明，保持证据唯一归属；
- 为高风险 Ticket 增加对抗式设计评审，并确保最终集成 Ticket 进入风险判断；
- 在第三轮修复前执行结构诊断，并由同一修复预算约束后续动作；
- 分离原生 Checkpoint envelope 与 Prepared Literal Payload，保证持久化命令按字面量安全传输；
- 将 Acceptance Seal 绑定到目标刷新、原生回读和修复预算，可靠处理目标漂移；
- 分离 Ticket Verdict、累计 PR/MR 审计与目标集成范围，补齐累计变更生命周期验证；
- 统一正式 Issue 标题前缀，减少 Tracker 语义漂移。

## 兼容性

- 20 个正式 Skill、11 个用户入口与 9 个模型可调用能力保持不变；
- 未新增状态机、Event、parser 或工作流 DSL；
- Spec、Ticket、Tracker、Git 与托管平台继续作为原生事实来源；
- 3.0.0 用户无需迁移工作项，可直接重新安装插件升级。

## 验证

- Release Validator、Runtime Contract、元数据与上游同步检查通过；
- GitHub、GitLab、Local 三种 Tracker 路径的 5 组 Fixture 通过；
- 142 项 Python 单元测试与 15 项显式 Agent 评测通过；
- 本地 Marketplace 重装、Installed Validator 与发布归档回读通过。
