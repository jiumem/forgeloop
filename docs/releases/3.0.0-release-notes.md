# Forgeloop 3.0.0

Forgeloop 3.0.0 将工程交付切换为 Tracker 驱动的 Spec/Ticket 模型，并提供 20 个边界清晰的 Codex Skills。

## 主要能力

- GitHub、GitLab、Local Markdown 三套等价 Tracker 路径；
- 单 Spec、多 Spec、Blocking Graph 与严格串行 Frontier；
- 独立 Coder、Spec Reviewer、Standards Reviewer；
- Role Task Pack、两轮有界修复、Branch/Integration Policy、冲突重审；
- Ticket、Spec、Initiative 三层最终验收；
- Multi-Spec 同 Commit 验收、成员延迟关闭与 owning-Spec Repair；
- 最小幂等 Checkpoint、成功/取消 Claim 释放与不依赖旧 child 的崩溃恢复；
- 11 个仅用户调用 Workflow 与 9 个模型可调用能力，触发描述统一聚焦 `Load when`；
- Codex Frontmatter、Invocation Policy、UI 元数据与插件 Manifest 完整校验。

## 破坏性变更

- `grill-initiative` 由 `grill-with-docs` 取代，大型探索使用 `wayfinder`；
- `plan-initiative` 收敛为 `to-spec` + `to-tickets`；
- `run-initiative-sequences` 合并到新 `run-initiative` 的多 Spec Initiative 路径；
- 新 `run-initiative` 只接受正式 Tracker Spec 或持久化 Initiative，不再读取或写入 `DESIGN.md`、`PLAN.md`、`LEDGER.md`；
- 不保留活动的 `legacy-*` Skill 或同义别名。

## 升级

这是破坏性版本。升级前请阅读 [2.5.0 → 3.0.0 迁移指南](../migrations/2.5.0-to-3.0.0.md)。
