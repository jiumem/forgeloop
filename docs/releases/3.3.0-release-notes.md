# Forgeloop 3.3.0

Forgeloop 3.3.0 将代码审查能力从固定 Git Diff 扩展为范围明确的已实现代码审查，并把正式 Skill 统一为 `$code-review`。

## 主要改进

- 支持 PR、固定 Base/Head、Commit 范围、工作区改动、指定文件、目录、模块和上游冻结候选；
- Git 只负责提供一种可复核范围，不再限制代码审查能力；
- Spec 与 Standards 两轴继续使用隔离上下文并独立报告 Findings；
- 没有 Spec 时保留 Standards 审查，并明确返回 `SPEC: NOT_AVAILABLE`；
- Description 明确排除代码调查、影响分析和调试，避免模型误加载；
- 用户可以直接调用，Agent 和 Workflow 也可以在实现范围冻结后自主调用。

## 不兼容变更

- `$review-change` 已删除并由 `$code-review` 取代；
- 不提供旧名称别名、兼容目录、Prompt 转发或自动回退；
- 自定义 Prompt、Workflow 和文档必须直接更新为 `$code-review`；
- 已加载 3.2.0 Skill 的旧任务不会原地升级，安装 3.3.0 后需新建任务。

## 架构边界

- 本次变更只调整 Agent Skill 提示词、元数据、活动引用与评测；
- 不新增状态机、Workflow DSL、运行时解析器或持久化事实源；
- 代码审查保持只读，不修改代码、测试、Git、PR/MR、Spec 或 Tracker；
- 20 个正式 Skill、11 个用户入口和 9 个模型可调用能力保持不变。
