# Forgeloop 3.6.0 发布候选

状态：发布前门禁与本地安装回读已通过，并取得用户发布授权；仅允许从 Release PR 合并后的最终 `main` Commit 发布。

## 发布范围

- `run-initiative` 的 Coder Role Task Pack 显式携带已批准的 Validation Entries、公共 Seam、Acceptance Prerequisites 和证据合同。
- 可复现的行为变更在修改生产代码前调用 `$tdd`，通过同一条仓库根目录公共命令保存真实 Red 和最终 Head Green。
- 新测试可以应用到冻结的 Base 生产代码形成 test-only Red；Red 必须由目标行为缺失导致。
- 行为保持型变更验证 Base/Head 公共行为与结构结果，不伪造 Red。
- 声明依赖外部条件的路径只接受批准的真实观察；条件不可用时阻塞，不使用 mock、recording adapter 或 harness 结论替代。
- `NO_CHANGE_REQUIRED` 保持 `Base == Head`、空 Diff 与现有行为证据合同。
- 代码修复重新进入适用的验证策略；只读 Repair Diagnosis 与 Exhaustion Diagnosis 不调用 `$tdd`。
- Scheduler 只验收策略与证据绑定的完整性和一致性；证据可信度由审阅同一固定 Head 的双 Reviewer 独立判断。
- harness 和 adapter 不得签发产品事实；决定性事实必须从产品边界产生并到达批准的公共 Seam。
- 不新增运行时状态、结果枚举、parser 字段、Tracker 状态或配置迁移。
- 20 个 Skill、11 个用户入口和 9 个模型可调用能力保持不变。

## 发布命令

只在 Release PR 合并且本地 `main` 与 `origin/main` 同步后执行：

```bash
release_commit="$(git rev-parse origin/main)"
gh release create v3.6.0 \
  --repo jiumem/forgeloop \
  --target "$release_commit" \
  --title "Forgeloop 3.6.0" \
  --notes-file docs/releases/3.6.0-release-notes.md \
  --latest
```

Tag 必须指向合并后的最终 `main` Commit；不得从本地候选提交或未合并分支发布。
