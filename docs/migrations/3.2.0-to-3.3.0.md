# Forgeloop 3.2.0 → 3.3.0 迁移指南

3.3.0 将 `$review-change` 断裂式替换为 `$code-review`。不提供旧 Skill、别名目录、Prompt 转发或运行时兼容分支。

## 必须修改

将自定义 Prompt、Agent 指令、Workflow 引用和文档中的：

```text
$review-change
```

替换为：

```text
$code-review
```

重新安装 3.3.0 插件并新建 Codex 任务，使新的 Skill 清单进入上下文。旧任务已经加载的 3.2.0 Skill 不会原地改名。

## 行为变化

`code-review` 审查范围明确的已实现代码，并保持 Spec 与 Standards 两轴独立：

- Git 范围：PR、固定 Base/Head、Commit 范围或工作区改动；
- 快照范围：指定文件、目录或模块；
- 上游范围：其他 Agent 或 Workflow 已冻结的候选实现与 Spec。

Git 只是范围输入，不再是能力边界。没有 Spec 时只运行 Standards 轴并报告 `SPEC: NOT_AVAILABLE`。

代码调查、影响分析、故障诊断和架构探索不会触发 `code-review`。分别使用普通代码探索、`$diagnosing-bugs` 或相应设计能力。

## 不变边界

- `code-review` 始终只读；
- Spec 与 Standards Reviewer 保持上下文隔离；
- 不新增 Tracker 状态、运行时解析器、状态机或持久化事实；
- 20 个正式 Skill、11 个用户入口和 9 个模型可调用能力数量不变。
