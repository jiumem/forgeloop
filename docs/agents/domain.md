# 领域文档

本仓库采用 single-context 领域文档布局。

## 探索代码库前

按需读取：

- 根目录的 `CONTEXT.md`；
- `docs/adr/` 中与当前工作范围相关的 ADR。

如果这些文件不存在，静默继续。不要仅为了满足流程预先创建空文件。

`$domain-modeling` 会在术语或持久决策真正得到确认时，按需创建或更新这些文件。

## 文件布局

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 术语规则

输出中的领域概念、Issue 标题、测试名称和设计提案应使用 `CONTEXT.md` 已定义的词汇，不得随意漂移到同义名称。

如果缺少所需术语，应先判断它是代码库并未采用的语言，还是需要通过 `$domain-modeling` 解决的真实领域缺口。

## ADR 冲突

如果方案与已有 ADR 冲突，必须明确指出冲突及重新讨论的理由，不得静默覆盖原决定。
