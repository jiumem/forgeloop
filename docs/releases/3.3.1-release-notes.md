# Forgeloop 3.3.1

Forgeloop 3.3.1 将代码审查 Skill 从 `$code-review` 更名为 `$spec-standards-review`，使机器名直接表达 Spec 与 Standards 双轴审查能力，并避免与 Codex 原生 Code Review 产品入口混淆。

## 变更内容

- Skill 目录、Frontmatter、Agent 元数据和默认 Prompt 统一改用 `spec-standards-review`；
- 所有活动 Workflow、Overlay、上游同步映射、测试和用户文档同步迁移；
- 保留完整双轴审查主体、隔离 Reviewer、范围冻结和只读边界，行为不变；
- `run-initiative` 将三轮 Repair Budget 明确为一次修复周期的诊断边界：耗尽后先确认暂停，再由 fresh Coder 基于完整语义证据建议自动续配、`IMPLEMENTATION_BLOCKED` 或 `CONTRACT_BLOCKER`；
- 自动续配保持同一 Ticket、Run 和 Branch，使用 fresh Coder 与 fresh 双 Reviewer，并通过现有原生 `RUN_RESUMED` 顺序解决并发恢复；不增加 replacement Ticket、Event、解析器或第二状态机；
- `CONTRACT_BLOCKER` 现在先形成完整调和包并完成内部语义 Review；一次用户决定可授权既有 `to-spec`、`domain-modeling`、`to-tickets` 完成原生调和，并在完整回读后暂停或竞争恢复原 Run；
- README、运行协议、fixture 与测试统一为“每周期三轮”，移除活动文档中的“两轮”矛盾；
- 20 个正式 Skill、11 个用户入口和 9 个模型可调用能力保持不变。

## 不兼容变更

- `$code-review` 已删除，不提供别名、兼容目录、Prompt 转发或自动回退；
- 自定义 Prompt、Workflow 和文档必须直接更新为 `$spec-standards-review`；
- 安装后需刷新 Skills 或重启 Codex，并在新任务中使用新名称。
