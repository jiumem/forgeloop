## 发布范围

- 将每组三轮 Repair Budget 定义为强制语义诊断边界，而不是 Ticket 的永久停止点。
- fresh Coder 在完整证据下执行 Exhaustion Diagnosis，可信 Scope 内新机制自动续配同一 Ticket、Run 和 Branch。
- 没有可信新机制或真实进展时保持 `IMPLEMENTATION_BLOCKED`，无条件继续不能恢复执行。
- 需要改变 Spec、Scope、Acceptance 或 ADR 时进入 `CONTRACT_BLOCKER`。
- 协调 Agent 在一次用户裁决前完成完整调和包与内部语义 Design Review。
- 一次批准覆盖 Spec/ADR 发布、Open Ticket 调和、完整原生回读以及原 Run 暂停或竞争恢复。
- 调和保持事实所有权、历史不可变和前向恢复，不新增 parser、状态机、Event、Reviewer 类型或第二事实来源。
- 20 个 Skill、11 个用户入口和 9 个模型可调用能力保持不变。

## 发布命令

只在 Release PR 合并且本地 `main` 与 `origin/main` 同步后执行：

```bash
release_commit="$(git rev-parse origin/main)"
gh release create v3.4.0 \
  --repo jiumem/forgeloop \
  --target "$release_commit" \
  --title "Forgeloop 3.4.0" \
  --notes-file docs/releases/3.4.0-release-notes.md \
  --latest
```

Tag 必须指向合并后的最终 `main` Commit；不得从本地候选提交或未合并分支发布。
